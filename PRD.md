| 항목 | 내용 |

---

## 1. 개요

전략팀 대시보드는 프로젝트 기반 조직의 인력·일정·휴가·가동률을 통합 관리한다. 간트 타임라인, 휴가 자동/수동 산정, Utilization 대시보드, 개인 CV, Engagement 검색을 제공하며 권한을 RLS로 강제한다.

## 2. 역할

| 역할 | 접근 |
|---|---|
| admin | 전체 편집 + 계정·권한·설정·백업 + Admin·Migration 메뉴(admin 전용) |
| editor | 전체 데이터 열람·편집 |
| **assistant** | *(v2.11 신설)* **지정된 화면(연차 관리 포함)에 한해 열람 전용** — 편집 컨트롤 비활성, 서버 RLS도 조회(SELECT)만 허용 |
| viewer | editor와 동일 화면을 보되 **전면 읽기 전용**. CV Generator·Leave는 **본인 정보만**, **Pipeline 비노출**, **Confidential 마스킹**, Admin·Migration 비노출 (§6.3) |

## 3. 데이터 모델 (요지)

| 테이블 | 주요 컬럼 |
|---|---|
| profiles | id, name, global_role, person_id(→people.id, 매칭키), lpn, status |
| people | id, lpn(unique), name, rank, role, hire_date, termination_date |
| work_items | id, type(project/proposal/pipeline), name, start, main_start, end_date, status(open/closed) — 전 유형, engagement_number, client, hashtags[], description, confidential(bool) (color 미사용, 유형 파생) |
| assignments | id, person_id, kind(work/leave), work_item_id, weekend_dates[], leave_type, start, end_date, note |
| accruals | id, person_id, direction(accrual/usage), type, days(+/-), date, source, note |
| leave_types | name, active(bool), sort_order |
| holidays | id, name, date, recurring |
| settings | key, value (fiscal_year_start_month=7) |
| audit_log | id, user_id, action, target_type, target_id, at |

> 입력 형식: Engagement Code `E-00000000`(E- + 숫자 8자리), LPN 숫자 5자리 `00000`. 계정↔인력 매칭은 `profiles.person_id`=`people.id`로만(LPN 미사용).

## 4. 색상 체계
- project=파랑, proposal=노랑, pipeline=회색, Leave=녹색. 각 계열 내 자동 음영. 수동 지정 없음. 유형 변경 시 색 자동 전환. **범례 표시는 T-2 참조(Leave는 세부 유형 나열 없이 통합 1항목).**

## 5. 기능 요구사항

### 5.1 실행취소 / 재실행
- Ctrl+Z=실행취소, Ctrl+Y(및 Ctrl+Shift+Z)=재실행. 최대 10단계. 대상: 배정·작업항목·인력·휴가 수동조정. 서버 역연산, 불가 시 안전 실패. editor/admin만. Undo/Redo 버튼.

### 5.2 타임라인 시각화
| ID | 요구사항 |
|---|---|
| T-1 | 인력별/작업별 뷰. 월→주→일 자동 전환. 모든 날짜 MM/DD. 반응형. 막대 라벨 폭 확장 + 호버 툴팁(작업명·고객사·기간, 인력별 뷰는 인력명). |
| **T-2** | 색상: project 파랑 / proposal 노랑 / pipeline 회색 / Leave 녹색. **범례는 4개 상위 카테고리(project/proposal/pipeline/Leave)만 표시한다.** 특히 **Leave는 지정휴가·프로젝트휴가·주말대체·포상·특별·지연보상·리프레시·휴직 등 유형별 세부 색상을 범례에 나열하지 않고, "Leave(녹색 계열)" 한 항목으로만 안내한다**(세부 유형별 음영 차이는 막대·호버 툴팁에서는 유지해도 되나 범례에는 노출하지 않음 — 항목이 너무 많아 가독성이 낮아지는 문제 수정). |
| T-3 | 작업 행은 그 작업 자체의 일정(span)만 표시(인원 막대 미중첩). 인원별 일정은 드롭다운으로. |
| T-4 | 작업 행·'휴가(전체)' 행 드롭다운 펼침/접힘. 작업 내 인원 직급순·이름순. |
| T-5 | 작업항목 더블클릭 → 상세 페이지(§5.7). |
| T-6 | Pre-study 구간 별도 분류·시각 구분(적립·CV 수행기간 제외). |
| T-7 | 인력 칩 드래그 배정(직급 그룹·이름순). **칩 목록에는 재직중 인원과 입사예정 인원을 포함하고 퇴사(P-1 기준) 인원은 제외한다.** 주말·공휴일 음영, 오늘 세로선/버튼, 주말 실근무 마커 날짜 고정, 정렬·필터·FY/기간 필터, 중복 배정 경고, sticky 헤더/라벨. |
| **T-8** | **헤더·본문 스크롤 동기화(버그 수정)**: 날짜 캘린더 헤더(상단)와 작업/인력 타임라인 본문(하단)이 **항상 동일한 가로 오프셋으로 함께 스크롤**되어야 한다. 특히 FY 필터로 조회 범위가 바뀌거나 좌우로 스크롤할 때 상·하단이 따로 움직여 날짜와 막대가 어긋나는 문제를 해결한다(공통 스크롤 컨테이너 또는 스크롤 위치 동기화). |
| **T-9** | **FY 다중 걸침 처리**: 한 작업/배정의 기간이 둘 이상의 FY에 걸쳐 있어도, FY 필터·좌우 스크롤 시 헤더 날짜와 막대 위치가 어긋나지 않게 한다(좌표 계산 기준을 단일 viewport 원점으로 통일). |

### 5.3 일정 편집
| ID | 요구사항 |
|---|---|
| E-1 | 빈 칸 드래그/더블클릭/칩 드롭으로 생성. 인력별 뷰 빈 칸 클릭 시 해당 인력 자동 선택. |
| E-2 | 작업 선택: status=open 만(전 유형). **인력: 재직중 또는 입사예정 인원만(퇴사 인원 제외, P-1 기준)**. 휴가 유형: active 만. |
| E-3 | 휴가 영업일 스냅. 드래그 겹침 방지 푸시(같은 인력 비중첩). |
| **E-5** | **Workitem 기간 자동 확장**: 소속 인력의 투입 **종료일이 workitem 종료일(end_date)보다 늦어지면 workitem의 종료일을 그 인력의 종료일로 자동 연장**한다. 반대로 인력의 투입 **시작일이 workitem 시작일(start)보다 앞서면 workitem의 시작일을 그 인력의 시작일로 자동 앞당긴다**(본 프로젝트 시작 main_start는 그대로 두되 start보다 앞설 수 없다는 무결성 유지). 배정 생성·이동·리사이즈·일괄 조작 어느 경로로든 동일 적용. Closed 작업은 잠금 규칙상 배정 변경 자체가 불가하므로 해당 없음. |
| E-4 | viewer는 모든 편집 컨트롤 비활성(서버 RLS로도 차단). |

### 5.4 인력 관리
- 추가/편집/삭제. 필드: LPN, 이름, 직급, 역할, 입사일, 퇴사일. 화면 필터·정렬.

| ID | 요구사항 |
|---|---|
| **P-1** | **재직 여부 실시간 파생(버그 수정)**: 재직 여부는 저장되는 값이 아니라 **조회 시점의 오늘 날짜(UTC 일 단위)와 hire_date·termination_date를 매번 비교해 동적으로 계산**한다. 규칙: 오늘 < hire_date → **입사 예정(비재직)**, hire_date ≤ 오늘 이고 (termination_date 없음 또는 오늘 < termination_date) → **재직중**, termination_date ≤ 오늘 → **퇴사(비재직)**. **현재는 레코드를 새로 불러와 저장하는 시점에만 termination_date를 비교해 분류하고 있어, 퇴사일이 지나도 재저장 전까지 재직자로 남거나 입사일이 안 지났는데도 재직자로 표시되는 버그가 있다.** 이 값을 저장·캐시하지 말고 매 조회·목록·필터·검색(E-2 인력 선택, LV-2/AL-2a 재직중 필터, §8 Utilization 등)마다 실시간 재계산해 사용한다. |
| **P-2** | **재직 상태 3단 구분을 전 필터에 반영**: 기존에는 "재직/퇴직" 2단으로만 구분하던 것을 **재직중 / 입사예정 / 퇴사** 3단으로 확장한다. 인력 관리(§5.4) 목록의 재직 여부 표시, 그리고 **재직중 필터를 제공하는 모든 화면(LV-2, AL-2a, CV Generator §5.9 등 공용 필터 컴포넌트를 쓰는 모든 곳)**에서 이 3단 구분을 선택할 수 있게 한다(예: 재직중/입사예정/퇴사 중 하나 이상 다중 선택). P-1의 판정 기준(오늘 날짜 실시간 비교)을 그대로 사용한다. |

### 5.5 작업항목 관리 & 상태(Open/Closed)
| ID | 요구사항 |
|---|---|
| W-1 | 유형 project/proposal/pipeline. 색상 선택 UI 없음(유형 파생·자동 전환). |
| W-2 | Open/Closed 상태를 전 유형에서 설정. Closed 작업은 타임라인·CV 표시되나 신규 배정 선택 제외. |
| W-3 | 필드: 유형, 이름, 전체 시작, 본 프로젝트 시작, 종료, 상태, Main Engagement No., Client, Description, 해시태그, Confidential. 목록 화면 필터·정렬(상태 포함). |
| **W-3a** | **목록 표에 Confidential 표시**: work items 목록(표)에서 confidential 처리된 항목은 **잠금/표식(예: 자물쇠 아이콘 또는 "Confidential" 배지)으로 기밀임을 한눈에 식별**할 수 있게 표기한다. |
| W-4 | Closed 전환 시 편집 잠금(전 역할): admin·editor에게도 읽기 전용. |
| W-5 | 편집하려면 Open으로 되돌려야 함. Open/Closed 전환은 editor/admin만. |
| W-6 | Closed 잠금은 서버(정책/트리거)에서도 강제(status를 open으로 되돌리는 전환만 허용). |

### 5.6 휴가 유형 관리
- leave_types.active 토글. 비활성 유형은 신규 휴가 배정 선택 제외(기존 이력 유지).

### 5.7 작업항목 상세 페이지
- 더블클릭 시 상세(유형·상태, 이름, 고객사, Engagement No., 기간(pre-study/본), Description, 해시태그, Confidential, 참여 인력 직급순·본 프로젝트 수행기간). editor/admin 편집·상태 전환(Closed는 W-4). viewer 마스킹.

### 5.8 Engagement 검색
- 작업명·고객사명·Description·해시태그 검색 → 참여 인력·해시태그 상세. confidential 마스킹. viewer도 동일(읽기 전용, Pipeline 비노출, 마스킹).

### 5.9 개인 CV (CV Generator)
- 인력별 수행 작업항목(Engagement No., Client, 본인 수행기간=본 프로젝트 한정, 해시태그). Open/Closed 무관. 수행 프로젝트만 필터(기본). Pipeline 제외. **이름 검색·필터 제공.** confidential 마스킹.
- viewer는 본인 CV만 조회·다운로드.

### 5.10 휴가 패널 (Leave)
| ID | 요구사항 |
|---|---|
| LV-1 | 적립/사용/잔여, 유형별 소계, 이력(차감 원천), 무급 내역. 사용·잔여 총량 정합(FIFO 원천 표시용). 수동 적립·사용 +/-. 자동 잔여 배정(가장 가까운 빈 영업일, 순서 ①주말대체→②프로젝트휴가→③포상→④지연보상, 유형 구분). |
| **LV-2** | **사람 검색·필터 제공(CV Generator와 동일)**: Leave 탭 상단에 인력 이름 검색 및 직급·재직상태 필터(**재직중/입사예정/퇴사 3단 구분, P-2 기준**)를 두어 대상 인력을 선택해 휴가 상세를 본다. |
| **LV-3** | **viewer 자동 상세창 잠김 버그 수정**: viewer가 Leave 탭에 들어가면 본인 휴가 상세가 자동 표시되되, **상세 창이 모달처럼 화면을 점유하거나 강제로 다시 열려 다른 탭 이동을 막는 일이 없어야 한다.** 닫기/탭 이동이 정상 동작해야 하며, viewer의 Leave는 본인 데이터를 **인라인(탭 본문)으로 표시**하고 강제 모달·자동 재오픈을 하지 않는다. |
| LV-4 | viewer는 Leave에서 본인 데이터만 열람, 편집 기능 비활성. |
| **LV-5** | **휴가 유형별 차감 원천 제한**: 사용 유형이 차감할 수 있는 적립 원천을 다음과 같이 제한한다. **특별휴가 사용은 오직 특별휴가 적립만을 원천으로 차감**한다(프로젝트휴가·포상휴가·주말/휴일대체·지연보상 등 다른 유형의 적립을 원천으로 사용할 수 없다). **지정휴가 사용은 프로젝트휴가·포상휴가·주말/휴일대체·지연보상 적립만을 원천으로 차감**하며(순서는 LV-1과 동일: ①주말/휴일대체→②프로젝트휴가→③포상→④지연보상), **특별휴가 적립은 지정휴가의 원천이 될 수 없다.** 수동 사용 입력·자동 FIFO 매칭 모두 이 제한을 따른다. |
| **LV-6** | **특별휴가 입력 단계 검증(버그 수정)**: 특별휴가는 적립이 먼저 존재해야 하며, 사용은 그 시점까지 누적된 특별휴가 적립 잔여 한도 내에서만 입력할 수 있다. **적립 잔여가 부족한 상태에서 특별휴가 사용을 저장하려 하면 입력 단계에서 차단**하고 부족분을 안내하는 에러 메시지를 보여주며, 저장을 허용하지 않는다. 특별휴가는 지정휴가와 달리 **선사용(적립 전 사용)을 허용하지 않는다.** (현재는 적립보다 사용을 먼저 입력해도 저장되어 차감 원천이 엉뚱한 유형으로 매칭되는 데이터 정합성 버그가 있음 — 본 검증으로 원천 차단.) |
| **LV-7** | **지정휴가 선사용 표시(버그 수정)**: 지정휴가는 적립이 부족한 상태에서도 사용 입력을 허용한다(선사용). 이 경우 차감 원천 표시를 모호한 "범용 N일" 등의 표기 대신 **"선사용 N일"** 로 명확히 표시하며, AL-7④ '지정휴가 선사용분'과 동일한 FIFO shortfall 기준을 사용한다. |

### 5.13 연차 관리 (Annual Leave, editor/admin 편집 · assistant 열람 전용 탭)

전략팀 자체 휴가 제도(§5.10 Leave)는 실제 휴가 운영의 기준이다. 연차 관리 탭은 그와 별개로 **회사의 공식 휴가 제도(법정연차·신입사원 휴가) 및 회사 타임시트와의 정합**을 위해 존재하며, 목적은 두 가지다: **(A) 퇴사 정산**, **(B) 타임시트 입력 판단을 위한 수치 안내**(코드 자동 매핑까지는 하지 않음 — 사람이 수치를 보고 판단).

| ID | 요구사항 |
|---|---|
| AL-1 | **viewer 접근 불가**(메뉴 비노출 + 서버 RLS 차단). **editor/admin은 열람·편집, assistant는 열람만 가능**(메뉴 노출, 적립 입력·보정 등 모든 편집 컨트롤 비활성, 서버 RLS도 SELECT만 허용). |
| AL-2 | **법정연차 적립(리필)**: FY와 무관하게 **매년 1월 1일 기준 역년(calendar year) 단위**. 사람마다 다르므로 **인력×연도별 적립량(법정연차/신입사원 휴가)을 수동 입력**한다. 자동 리필 계산 없음. 수동 보정(+/-, 사유 기재) 제공. |
| AL-2a | **인력 검색·필터**: 연차 관리 탭(퇴사 정산 뷰·수치 안내 뷰 공통) 상단에 **Leave 탭(LV-2)과 동일한 공용 필터 컴포넌트**를 재사용하여 **이름 검색 + 직급 필터 + 재직 상태 필터(재직중/입사예정/퇴사 3단 구분, P-2 기준)**를 제공한다. 검색·필터로 좁힌 목록에서 대상 인력을 선택해 상세를 연다. assistant도 조회 목적으로 동일하게 사용 가능. |

**(A) 퇴사 정산 로직**

핵심 원칙: 팀 제도로 정당 적립된 휴가(프로젝트휴가 등)와 법정연차는 **합산되는 별개 권리가 아니라, 둘 중 더 큰 쪽이 총 휴가 권리**가 된다. 단, **주말/휴일대체 휴가**(실제 근무에 대한 법정 성격의 보상)와 **특별휴가**(예비군·경조사 등 연차와 별개로 발생하는 권리)는 법정연차 쪽 후보에도 함께 더해서 비교한다. 즉 구성원은 max(법정연차 누적 + 주말/휴일대체 누적 + 특별휴가 누적, 팀 정당 적립 누적) 이상의 휴가 권리를 가지며, 그 범위 내 사용은 퇴사 시 불이익이 없다. **무급 휴가는 이 비교의 어느 쪽에도 포함하지 않는다.**

| ID | 요구사항 |
|---|---|
| AL-3 | **총 휴가 권리(Entitlement)** = **max( 법정연차 누적 합(AL-2, 보정 포함) + 주말/휴일대체 누적 적립 합 + 특별휴가 누적 적립 합, 팀 정당 적립 합(프로젝트휴가·주말/휴일대체·포상·특별·지연보상 등 Leave 탭의 적립분 전체) )**. **주말/휴일대체·특별휴가는 팀 정당 적립 합에도 이미 포함되어 있으나, 법정연차 쪽 후보 계산에도 별도로 더해서 두 후보값을 비교**한다(어느 한쪽 합계에 두 번 차감되는 것이 아니라, max 비교를 위한 두 후보 각각에 정상적으로 포함되는 것). 법정연차+주말/휴일대체+특별휴가 합이 팀 정당 적립 합보다 크면 그 값을 총 휴가 권리로 한다. **무급 휴가는 적립이 존재하지 않으므로 이 계산에 포함되지 않는다.** |
| AL-4 | **초과 사용분(퇴사 시 차감)** = max(0, 총 유급 사용일수 − 총 휴가 권리(AL-3)). 정당 권리 범위 내 사용은 차감하지 않는다. **총 유급 사용일수에는 무급 휴가 사용을 포함하지 않는다**(무급은 애초에 적립·권리 계산과 무관). |
| AL-5 | **미달 보상분(퇴사 시 보상)** = max(0, 총 휴가 권리(AL-3) − 총 유급 사용일수). 권리보다 적게 썼으면 그 차이만큼 보상한다. 이 계산에서도 **무급 휴가는 사용일수에 포함하지 않는다.** |
| AL-6 | **퇴사 정산 뷰**(상세 내역 필수 표시, 합계 숫자만 아님): 인력·기준일(퇴사일) 선택 시 다음을 표로 표시한다. ① **법정연차 적립·보정 "내역"**(연도별 적립일수 + 보정 항목 각각과 사유를 **펼친 리스트**로), ② **팀 정당 적립 "내역"**(유형별 적립 일수와 발생 원천을 **펼친 리스트**로 — 주말/휴일대체·특별휴가 항목 포함), ③ **총 휴가 권리 비교**: **(a) 법정연차+주말/휴일대체+특별휴가 = X**, **(b) 팀 정당 적립 합 = Y**를 나란히 보여주고 **총 휴가 권리 = max(X, Y)**, 어느 쪽(a/b)이 기준이 되었는지 표시, ④ **총 유급 사용 "내역"**(사용 일자·기간·휴가 유형·차감 원천을 포함한 **펼친 리스트**로, **무급 사용 건은 이 리스트·합계에서 제외**하고 참고용으로만 별도 표시 가능), ⑤ 초과 사용분(AL-4), ⑥ 미달 보상분(AL-5), ⑦ 최종 정산(⑥−⑤). **⑤~⑦의 합계 수치는 ①②④ 리스트의 합계와 정확히 일치해야 하며 별도 재계산하지 않는다.** 퇴사자(termination_date 보유) 조회 가능. |

**(B) 타임시트 판단용 수치 안내**

일자별 코드 자동 매핑은 제공하지 않는다. 대신 담당자가 판단할 수 있도록 **당일(조회 기준일) 기준 누적 수치**를 요약해 보여준다.

| ID | 요구사항 |
|---|---|
| AL-7 | **수치 안내 뷰**: 인력·기준일 선택 시 다음을 표시한다. ① **법정연차/신입사원 휴가 누적치**(해당 역년 적립 합 + 보정), ② **프로젝트휴가 기 사용분**(기준일까지), ③ **지정휴가 중 프로젝트휴가 적립분에서 차감된 기 사용분**(FIFO 차감 원천 기준), ④ **지정휴가 선사용분**(적립 부족 상태에서 초과 사용한 일수, FIFO shortfall 기준). 이 수치들을 근거로 담당자가 타임시트 입력 코드를 판단·안내한다. |
| AL-8 | 수치는 Leave 탭의 배정/적립/FIFO 계산 결과를 읽어 산출한다(중복 저장 없음). 역년 경계(1/1 리필)를 반영한다(이월 없음 기본, 예외는 수동 보정). |
| AL-9 | 데이터: `annual_leave_grants`(person_id, year, days, note), `annual_leave_adjustments`(person_id, direction, days(+/-), date, note). audit_log 기록. |
| AL-10 | **퇴사 정산서 HTML 내보내기**: 퇴사 정산 뷰에 **"HTML로 저장"** 버튼을 추가한다. 클릭 시 AL-6의 ①~⑦ 전체(적립 내역·사용 내역 상세 포함)를 담은 **단일 인쇄용 HTML 파일**을 다운로드한다. 문서 상단에 인력명·직급·입사일·퇴사일·정산 기준일을 표기하고, **"법정연차 적립 내역" → "팀 정당 적립 내역"(유형별) → "휴가 사용 내역"(일자별) → "정산 요약"** 순서로 섹션을 명확히 구분한다. 정산 요약은 굵게 강조하고 위 세 섹션의 합계와 정확히 일치시킨다. CV Generator(§5.9)의 HTML 다운로드·인쇄 친화 스타일을 재사용한다. editor/admin과 assistant 모두 다운로드 가능(assistant는 편집 불가, 다운로드는 조회 동작으로 간주). |

### 6.3 역할별 접근
| 역할 | 접근 |
|---|---|
| admin | 전체 편집 + 계정·권한·설정·백업 + Admin·Migration |
| editor | 전체 데이터 열람·편집 |
| **assistant** | *(v2.11)* 지정된 화면(연차 관리 포함) **열람 전용**(편집 비활성, RLS로 쓰기 차단) |
| viewer | editor와 동일 화면 **전면 읽기 전용**. ① 모든 편집 비활성(서버 RLS 차단), ② CV·Leave는 본인(person_id)만, ③ Pipeline 비노출, ④ Confidential 마스킹, ⑤ Admin·Migration 비노출 |

### 6.4 Closed 잠금(전 역할)
- Closed 작업/배정은 admin·editor에게도 편집 비활성. Open 전환 후 편집(§5.5). 서버 강제.

### 6.5 RLS 요지
- people: 인증 전 역할 SELECT 전체(쓰기 editor/admin). work_items: viewer는 type≠pipeline만(마스킹 work_items_safe). assignments: viewer는 pipeline 연결분 제외. accruals: viewer는 본인만. 전 테이블 쓰기 editor/admin + Closed 잠금. **`annual_leave_grants`/`annual_leave_adjustments`: assistant는 SELECT만 허용(전 인력 대상), INSERT/UPDATE/DELETE는 editor/admin만.** (부록 B)

## 7. 휴가 산정 규칙
1. 프로젝트휴가: **인력×프로젝트 단위 합산 적립** — 한 인력이 같은 프로젝트에 둘 이상의 분리된 배정으로 투입된 경우, 배정별로 각각 round 적립하지 않고 **해당 프로젝트 내 모든 투입 구간(각각 본 프로젝트 구간과의 교집합)의 달력일수 합**을 먼저 구한 뒤 **round(합계 투입일수/10)** 로 1회 적립한다. Pre-study 제외. (예: 12일+9일로 나뉜 투입 → round(21/10)=2일. 배정별 계산 round(12/10)+round(9/10)=1+1과 다름에 유의.)
2. **주말/휴일대체**: 실근무일만 인정. **주말(토요일·일요일 모두 포함)** 1일당 **0.5일**, **공휴일** 1일당 **1.0일** 적립(pipeline 제외). **주말과 공휴일이 겹치는 날(예: 공휴일이 토·일요일인 경우)은 주말 기준(0.5일)을 우선 적용**하며, 공휴일 가산을 별도로 중복 적용하지 않는다.
3. **지연보상(수동 전환)**: 지연보상은 **자동 산정하지 않는다**(기존 자동 판정 로직 제거). 프로젝트 종료 후 휴가 사용 지연이 자의인지 회사 사정인지는 시스템이 판단할 수 없으므로, **관리자가 휴가 패널의 수동 적립으로 '지연보상' 유형을 직접 부여**한다(권장 기준 참고치: 적립분 ≤1:0 / 1.5~3:1 / 3.5~5:2 / 5.5↑:3 — UI 도움말로만 안내). |
4. 사용·잔여 총량 정합(FIFO 원천 표시용, 음수 허용). 수동 +/-.
5. 자동 잔여 배정: §5.10.
6. 휴가 영업일 스냅.
7. **소급 재계산(v2.13 버그 수정 적용)**: 위 규칙2 변경 시점 이전에 "주말+공휴일 겹침일을 공휴일(1.0일)로 과다 적립"했거나 "일요일을 주말로 인식하지 못해 0.5일을 누락 적립"한 기존 accruals 이력을 **일회성 마이그레이션으로 자동 재계산·정정**한다. 별도 관리 화면·미리보기·확인 UI 없이, 배포 시 한 번 실행되는 스크립트가 대상 건을 찾아 규칙2 기준 올바른 값과의 차액만큼 보정(+/-) 항목을 자동으로 적립한다(기존 행은 삭제하지 않고 보존). 사유는 "v2.13 주말/공휴일 겹침 산정 버그 소급 정정"으로 기재하고 audit_log에 기록한다.
